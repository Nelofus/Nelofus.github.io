    1. CF1826E. Walk the Runway

      高维偏序。考虑 std::bitset 直接做，时间复杂度 \(\mathcal O(\frac{n^2m}{w})\)。

      实现的时候可以枚举维度，对这一维排序，维护所有当前小于它的位置集合 \(\mathrm{cur}\)，把等于它的所有数的集合交上 \(\mathrm{cur}\)。

    2. CF1749E. Cactus Wall

      一个墙当然是一条斜着延伸的从左到右的折线。

      每个点向四个斜着的点连边，点权有仙人掌为 \(0\)，否则为 \(1\)，\(S\) 连左边一列，右边一列连 \(T\)，跑最短路，最短路就是这个墙。

      注意旁边有仙人掌的点是被禁用的，点权设为 \(\text{inf}\) 或者不连都行。用 \(01\text{BFS}\) 容易做到线性。

    3. CF946F. Fibonacci String Subsequences

      考虑全落在 \(i-1\)，\(i-2\) 和横跨 \(i-1,i-2\) 的情况。

      设 \(f_{i,l,r}\) 表示在 \(F(i)\) 中，原串子串 \([l,r]\) 的出现次数之和。转移平凡。

      注意到中间转移的过程与当前状态无关，其实可以矩阵快速幂 \(\mathcal O(n^3\log x)\) 的（需要 \(f_{i,l,r}\) 代表的子串改成 \([l,r)\)）。但是写了个暴力递推 \(\mathcal O(n^3 x)\)。

    4. CF1617D2. Too Many Impostors (hard version)

      考虑 D1 的 \(2n\) 怎么做。

      不妨假设我们已经知道了两个点的值 \(p_1,p_2\)，并且他们 \(val_{p_1}\) 是 \(0\)，\(val_{p_2}\) 是 \(1\)，那么通过 \(n\) 次询问就能够找到每个点是 \(0\) 还是 \(1\)。

      那再考虑这两个点的值要怎么找，我们问 \((i,i+1,i+2)\) 和 \((i+1, i+2, i+3)\)（这里 \(>n\) 就减 \(n\)，形成环）如果他们的值不同那么 \(i,i+3\) 肯定一个是 \(0\) 一个是 \(1\)。并且与 \(val_i=\operatorname{ask}(i,i+1,i+2),val_{i+3}=\operatorname{ask}(i+1,i+2,i+3)\)。

      考虑优化：

      我们分成 \(n/3\) 块询问，找到一个为 \(0\) 的块和一个为 \(1\) 的块，前者起点为 \(p\)，后者起点为 \(q\)。

      那么询问 \((p+1,p+2,q)\) 和 \((p+2,q+1,q+2)\)，\(p,p+1,p+2,q\) 这四个点为起点的询问必定是有一个 \(0\) 和一个 \(1\) 的，并且相邻询问都有两个相同的值，这样就能找到 \(0\) 和 \(1\) 了。这样消耗了 \(\frac{3}{n}+2\) 次询问。

      接下来对每个块花 \(2\) 次询问问出答案。包含 \(p_1,p_2\) 的块直接暴力问即可。否则：

      对每个块，设起点为 \(s\)，如果结果为 \(0\)，那么去掉 \(s+2\) 后有两种情况，即 \((0,0)\) 和 \((0,1)\)。

      后者可以确定出 \(s+2\) 肯定是 \(0\)，前面两个恰一个 \(0\) 和一个 \(1\)，前者可以一下确定出两个元素，最后一个直接问。

      只要询问 \((p_2,s,s+1)\)，如果是 \(0\) 则为前者，如果为 \(1\) 即为后者。

      对于 \(1\) 的情况，对称一下。

      总询问次数 \(n+2\)。

    5. CF1442C. Graph Transpositions

      注意到反转的代价增长的很快，只需要 \(\mathcal O(\log_2 m)\) 次，就会无论如何都优先减小翻转次数。

      那么前 \(\log\) 次跑分层图最短路，然后最短路的时候先最小化翻转次数，再最小化路径长度即可。

      考虑能不能写得更阳间一点，一个点 \(u\) 可能成为最优解的翻转次数区间是 \([least_u,least_u+\lceil\log m\rceil]\)，其中 \(least_u\) 是到达 \(u\) 至少要翻转几次。那么设 \(f_{u,k}\) 表示到达点 \(u\) 时翻转次数是 \(least_u+k\) 次，忽略翻转代价的最小值。即可简单计算。

      笑点解析：

      auto &[u, k] = q.front();
q.pop();

    6. CF1004E. Sonya and Ice Cream

      我们知道所有直径中点相同，一个事实是，我们删去中点把树分割成若干个连通块，每个联通块的里的所有点 \(u\)，到直径中点和落在它那个连通块的直径端点（如果有的话）的距离 \(\max\)，一定大于它到任何一个连通块里的其他的点。

      所以，长度为 \(k\) 的链的两个端点一定都落在直径上。

      我们抽出任意一条直径，求出直径上的每个点 \(u\) 在不走直径任何一个点情况下的最大距离，记为 \(val_u\) ，那么每个长度为 \(k\) 的链的权值就是，链中每个点的 \(val_u\) 的 \(\max\)，与链顶和链底分别到它们那一侧的直径端点的距离取 \(\max\)。

      前面那个东西用单调队列就线性了。

      Submission #266206212

    7. CF1493F. Enchanted Matrix

      第一步显然是枚举质因子，然后一个个尝试删掉。

      考虑一个质因子 \(p\) 能否被删掉，那就是问能不能把当前的 \(n\) 行再分成 \(p\) 段，变成 \(\frac{n}{p}\) 行。

      下面的区间指的都是段的区间。

      如果没有不能重叠的要求，经典地，询问 \([1, p-1]\) 和 \([2,p]\) 即可。

      显然有了不能重叠的要求，一个朴素的想法是把这 \(p\) 段分成两段，问前面和后面是否相等。

      如果 \(p\) 是 \(2\)，那可以直接问。

      否则设 \(q=\lfloor\frac{p}{2}\rfloor\)，考虑把经典的做法和朴素想法结合起来，我们询问 \([1,q]\) 分别与 \([q+1,2q]\) 和 \([q+2,2q+1]\) 是否相同，如果都相同，那么 \([q+1,2q]\) 与 \([q+2,2q+1]\) 相同，所以 \([q+1,2q+1]\) 全部相同，从而 \([1,2q+1]\) 全部相同。

      如果有不同的，显然不可能。

    8. CF1648B. Integral Array

      记事本未免有点太难用了。

      注意到 \(\sum C\) 是有保障的。

      枚举不在集合中的 \(t\) 表示整除的结果，对每个集合中的 \(y\)，\(\lfloor\frac{x}{y}\rfloor=t\) 代表着 \(x\in [ty,t(y+1))\)。

      对每个 \(y\) 检查这个区间里面是否有数即可，复杂度调和级数 \(\mathcal O(C\log C)\)。

    9. CF1651F. Tower Defense

      考虑这个过程，每次会令一个前缀 \(=0\)，最后一个位置可能 \(\neq 0\)，那就单独分一段长度为 \(1\) 的。

      这个过程其实就是一个前缀的推平，现在每个长度 \(\ge 2\) 的段，所有起始值都是 \(0\)，并且起始时间相同，长度为 \(1\) 的段处理是简单的，直接考虑原来剩余的，加上时间的差 $\times $ \(r\)，再和 \(c\) 取 \(\min\) 就是当前值。

      因为起始时间相同并且全 \(0\)，所以我们记现在和上一个时间差为 \(difft\)。

      对每个点 \(i\)，分 \(difft< \lceil\frac{c_i}{r_i}\rceil\) 和 \(difft\ge \lceil\frac{c_i}{r_i}\rceil\) 讨论，前者是 \(difft\cdot r_i\)，后者就是 \(c_i\)。

      对每个时间 \(p\)，所有点的贡献形如一些一次函数之和，若 \(p<\lceil\frac{c_i}{r_i}\rceil\)，则它的贡献是 \(r_ix+0\)，如果 \(p\ge \lceil\frac{c_i}{r_i}\rceil\)，那它的贡献就是 \(0x+c_i\)。我们要查询一个区间和，就是把这些一次函数加起来，再把 \(p\) 带入进去就行了。

      注意到 \(\max t\le 2\times 10^5\)，所以我们开 \(t\) 棵线段树，维护每个时刻的函数。这个过程可以可持久化。

      段合并的过程就是对每个段先算和是否 \(>h\)，如果 \(>h\) 就二分找到第一个点满足前缀和 \(>h\) 的，然后把这个点单独成一段，再把前面抹平。否则直接令 \(h\) 减去这段和，跳过这段。

      用线段树上二分能做到 \(\mathcal O(n\log n+t+q\log n)\)。

      代码重构了一次，参考了这东西，写起来非常怄火，但是写完很快就过了。

      Submission #266530453，因为不是很会在一个区间的线段树上二分，所以这个部分是自己脑补的。
